# 능동 코디네이션 프로토콜 (Codex)

> **적용 범위: 대화형 팀(로컬 lead + 사이드카) 경로.** 대량·독립·결정론 작업으로 분기됐다면(batch 경로) 스크립트 내부 adversarial-verify 스테이지가 검증을 담당하므로 이 프로토콜을 쓰지 않는다.
> 기본은 **적응형 실행**. 의견 수렴이 필요한 갈림길(게이트)에서만 "서로 안 보는 독립 의견 + 심판 1명"을 부른다.
> 여럿이 라운드로 서로 반박하는 토론은 기본이 아니다 — 같은 모델끼리라 바이어스(상관된 의견 + 아첨)로 가짜 합의가 나기 때문.

---

## 0. 한 장 요약

```
[척추] 적응형 실행 — 항상 기본. 모드 선택 없음.
   ├─ 평소: 그냥 실행 직진
   └─ 갈림길(게이트) + 조건 충족 시만:
        독립 의견 N개(서로 안 봄, 다른 근거/모델) → 심판 1명이 차이 정리
          ├─ 표현차일 뿐  → 즉시 결정, 복귀
          └─ 진짜 충돌 + 되돌리기 비쌈 → 딱 1회 타깃 반박 → 결정, 복귀
        (스코프 한정 · 예산 상한 · 결과만 실행에 반영 · 게이트가 게이트 못 부름)
```
**검증 가능한 건 토론 말고 도구로 확인한다.** 토론은 검증 불가능한 가치판단에만.

---

## 1. 원칙 — 왜 독립 의견 + 심판인가

- **같은 base 모델 N명 = 상관된 노이즈.** "의견이 모인다"가 진리 수렴이 아니라 같은 prior로의 쏠림(가짜 합의).
- **라운드 반박은 바이어스를 키운다.** 서로의 글을 보고 반응하면 아첨·쏠림으로 가짜 합의가 *더 빨라진다*. → "독립"(서로 안 봄)이 바이어스를 죽이는 핵심.
- **자기보고 수렴 지표는 못 믿는다.** AI는 자기 confidence를 정확히 못 매기고, "입장 바꿈"을 꾸며낸다.
- **비용도 안 맞는다.** 다라운드 ≈ 순차 15콜대 vs 독립 의견+심판 ≈ 4콜(병렬), 이득은 비슷.

→ **토론은 기본이 아니라 켜는 쪽이 입증책임을 지는 예외.** 대부분 작업의 토론 횟수 = 0.

---

## 2. 척추 — 적응형 실행 (drive → inspect → re-inject)

```
[셋업] 팀원에게 태스크 배분.
[루프] 팀원 보고가 도착할 때마다:
    팀장이 보고 검토(검증 / 모순 / 공백):
       ├ 충분        → 다음 태스크 배정
       ├ 약함·모순   → 즉시 재지시  또는  교차검증 위임
       └ 새 필요 발견 → 런타임에 새 전문가 스폰 (예: "DB 병목 발견 → 인덱스 전문가 추가")
    종료: 모든 태스크 완료 + 검증 통과
[종합] 통합 리포트
```
- **런타임 스폰** — 계획에 없던 전문가를 결과를 보고 추가하는 적응. (결정론 batch 스크립트는 분기가 미리 박혀야 해서 못 함.)
- 팀장은 **collect-at-end 금지** — 중간 산출을 읽고 그때그때 조정.

> Codex 주의: 팀장(이 스킬을 실행 중인 Codex)이 로컬에서 루프를 직접 돈다. 사이드카 에이전트의 최종 노트를 읽고 → 다음 수를 정한다. 한 번 스폰하고 끝까지 안 보는 fire-and-forget 금지.

---

## 3. 게이트 — 독립 의견 + 심판

### 발동 조건 (아래 *복수* 동시 충족 시만)
1. 살아남은 선택지가 **2개 이상**
2. **되돌리기 비쌈** (스키마·인증·돈·공개 API 등 폭발 반경 큼)
3. **도구 1번으로 사실 확인 불가** (검색·조회로 끝나면 게이트 말고 그냥 조회)
4. **가치 충돌 실재** (비용 vs UX vs 보안이 실제로 다툼)
5. **최초 1회만** (이미 내린 결정은 공유 메모리에 캐시 → 재토론 금지)

→ 하나라도 약하면 **실행 직진**.

### 절차
```
1. 독립 의견 N개(2~3) 병렬 수집 — ★서로 안 봄(블라인드). 교차 주입 없음.
   가능하면 서로 다른 근거(도구/자료 배정) 또는 다른 모델(로컬 / Codex CLI / agy).
2. 심판 1명이 의견 diff → 분류: (a) 표현차일 뿐  (b) 실재 충돌
3. (a) 표현차 → 즉시 결정 + 실행 복귀
4. (b) 실재 충돌 AND 비가역 → 그 축 하나에 대해 ★딱 1회 타깃 반박
       (2명, 그 쟁점만, 익명·근거 기반) → 결정 + 복귀
5. 결과를 실행 루프에 '제약'으로 기록. 게이트는 스코프 한정 · 예산 상한 · 재귀 금지.
```

---

## 4. 다양성은 "근거"에서 (성격 부여 금지)

- 같은 모델 N명은 블라인드여도 상관됨 → 진짜 독립은 **서로 다른 근거**:
  - 서로 다른 도구/자료 배정 (A=문서 조회, B=코드 grep, C=테스트 실행) → 결론이 갈리면 근거가 달라서 갈린 것.
  - 가능하면 **다른 모델** (로컬·Codex CLI·agy 혼합). 같은 모델 여럿보다 정보량 큼.
- ❌ 프롬프트로 성격/페르소나만 다르게 = 가짜 다양성.
- ❌ devil's advocate 강제 = "역할극"으로 사전 디스카운트됨.

---

## 5. 수렴/종료는 외부 신호로 (자기보고 금지)

- ❌ 자기보고 confidence / "입장 바꿈" 플래그.
- ✅ 외부 신호: **두 안의 실제 diff**(코드/스펙/테스트 결과), **도구 검증 통과**, "새 논점이 더 안 나옴".
- 이견이 안 풀리면 **억지 합의 금지** — "잔존 이견 명시"하고 심판/팀장이 결정.

---

## 6. 검증 가능한 건 토론 금지

사실 문제(누가 맞나)는 **도구·검색·테스트로** 해소. 토론은 *검증 불가능한 가치판단*에만.

---

## 7. 비용 가드

- 게이트 1회 = 독립 의견 2~3콜 + 심판 1콜 (+필요 시 반박 2콜). **다라운드 금지.**
- 게이트 토큰 상한(작업 예산의 일정 %). 초과 시 팀장 단독 결정 + 미결 질문 로깅.
- 게이트는 작업당 드물게. 입증책임은 켜는 쪽.

---

## 8. 팀장 역할 (가볍게)

- 팀장 = 라우터 + 심판 호출자. **"가장 강한 반론 선정" 같은 편집 판단 최소화.**
- 독립 의견을 팀장이 요약해 섞지 말 것 — 원문 그대로 심판에 넘긴다.

---

## 안티패턴
- ❌ 다라운드 반박 토론을 기본으로
- ❌ 자기보고로 수렴 판정
- ❌ 성격 부여 / devil's advocate 강제로 다양성 흉내
- ❌ 검증 가능한 사실을 토론에 부치기
- ❌ 게이트가 게이트를 재귀 호출 / 게이트→실행→게이트 핑퐁 (재귀 깊이 1)
- ❌ 팀장이 collect-at-end로 끝내기 / 사이드카 fire-and-forget
