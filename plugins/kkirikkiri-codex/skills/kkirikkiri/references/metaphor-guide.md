# 용어 가이드 — 공식 용어 + 한글 설명 병기 (Codex)

> 끼리끼리는 비개발자도 사용한다. 그렇다고 공식 용어를 메타포로 **대체하지 않는다.**
> **원칙: 용어·기능명은 공식 명칭을 그대로 쓰고, 한글은 설명으로만 병기한다.**
> 표기 형식: `공식 용어 (한글 설명)` — 예: "Opus (가장 똑똑한 모델)", "Codex CLI (OpenAI 코드·대규모 분석 도구)"

---

## 모델 / 외부 CLI (공식 용어 그대로 + 설명 병기)

| 공식 용어 (그대로 표기) | 병기할 한글 설명 | 언제 쓰는지 |
|----------|---------------------|------------|
| Opus | "가장 똑똑한 모델" | 팀장, 분석·비평·종합, 핵심 구현 (판단 무게 높은 역할) |
| Sonnet | "균형형 모델" | 일반 워커 (리서치·드래프트·간단 구현) — 기본값 |
| Haiku | "경량·고속 모델" | 기계적 잡일 한정 (수집·포맷·추출) |
| Codex CLI | "OpenAI 코드·대규모 분석 도구" | 코드 분석/리뷰 + 대규모 분석. cross-model 검토 1순위 |
| Antigravity CLI(agy) | "디자인/UI 도구" | 디자인/UI (Gemini CLI 후속 — Gemini는 언급하지 않음) |
| gajae-code(gjc) | "멀티모델 코드·교차검토 도구" | 코드 구현·분석 + cross-model 검토 |

> Codex 주의: "Opus 티어 / Sonnet 티어"는 *역할의 판단 무게* 분류일 뿐이며 Codex 런타임의 모델 핀이 아니다. 사용자에게 보여줄 땐 공식 용어 + 한글 설명 병기 형식만 지키면 된다.

### 예시
```
❌ "가장 똑똑한 AI가 팀장을 맡아요"            (용어 은닉 — 금지)
✅ "팀장은 Opus(가장 똑똑한 모델)가 맡아요"

❌ "전문 AI 2명이 리서치를 담당해요"
✅ "Sonnet(균형형 모델) 2명이 리서치를 담당해요"

❌ "코드 전문 AI가 코드를 검토해요"
✅ "Codex CLI(OpenAI 코드 분석 도구)가 코드를 검토해요"
```

## 실행 방식 (substrate — 공식 용어 그대로)

| 공식 용어 (그대로 표기) | 병기할 한글 설명 |
|----------|---------------------|
| 대화형 팀 (로컬 lead + 사이드카) | "실시간 협업 팀" — 팀원들이 의견을 주고받으며 토론·수렴 |
| batch (결정론 병렬 처리) | "대량 자동 병렬 처리" — 수십 개 작업을 병렬로, 재실행 가능 |

```
❌ "작전 통제실로 진행할까요?"                 (만든 메타포로 기능 대체 — 금지)
✅ "대화형 팀(실시간 협업 팀)으로 진행할까요?"
```

## 내부 구현 (언급하지 않음 — 사용자가 알 필요 없는 배관)

| 내부 용어 | 대신 쓸 표현 |
|----------|---------------------|
| spawn_agent | "팀원을 불러오다" |
| shutdown / 종료 | "작업 종료" |
| TEAM_PLAN.md / TEAM_PROGRESS.md / TEAM_FINDINGS.md | "작업 계획/진행/발견 기록 파일" — 내부 파일명 비노출 |
| DEAD_ENDS | "실패한 시도 기록" |
| Ralph Pattern / 검증 루프 | "자동 재시도" (커뮤니티 패턴명 — 공식 용어 아님) |
| preset | "팀 템플릿" (끼리끼리 자체 개념) |
| adversarial-verify | "교차 검증 단계" |
| archetype | "역할 유형" (끼리끼리 자체 개념) |

> 단, **사용자가 직접 물어보면** 내부 용어도 숨기지 않고 솔직하게 설명한다.

## 일반 원칙

1. **공식 용어를 메타포로 대체하지 않는다** — Opus, Sonnet, Haiku, Codex, agy, gjc, MCP는 그대로 표기
2. **용어만 던지지도 않는다** — 반드시 `공식 용어 (한글 설명)` 형식으로 병기
3. **내부 구현(배관)은 노출하지 않는다** — spawn_agent, 내부 파일 경로 등. 단 사용자가 물어보면 솔직히 설명
4. **에러 메시지는 번역한다** — 기술적 에러 → "문제가 생겨서 다른 방법으로 처리했어요" + 원인 한 줄
